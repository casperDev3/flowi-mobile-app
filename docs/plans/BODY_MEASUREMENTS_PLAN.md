# План: щоденні записи тіла (вага + заміри) + нагадування

**Дата:** 2026-06-30
**Запит:** на вкладці «Здоров'я» — додавання щоденних записів: вага, заміри тіла (груди, талія, стегна тощо) + усе корисне для контролю, і **нагадування про внесення даних**.

---

## 1. Як це лягає в поточну архітектуру

- Усі дані здоров'я вже в одному масиві `health_entries_v2: HealthEntry[]` з дискримінатором `type`. Виміри — **просто нові `type`** (як `weight`/`pulse`), що дає безкоштовно: збереження, `lastForDay`, 7-денні чарти, історію, бекап, sync.
- Хук `use-health-entries.ts` — єдине джерело; додамо агрегати вимірів.
- `HealthEntryModal` — для разового вводу; для тіла зробимо **комбіновану форму** (усе за раз).
- Нагадування: вже є `scheduleDailyReminder`; додамо `scheduleWeeklyReminder` і розширимо модель `Reminders`.
- Хаб має плитку **«Показники тіла»** (`/health-vitals` = вага + пульс).

**Висновок:** мінімум нових абстракцій — переважно нові `EntryType` + один екран + одна форма + нагадування.

---

## 2. Модель даних

### 2.1. Нові `EntryType` (значення в см, % жиру — у %)
```ts
// додати до EntryType:
| 'chest'     // груди
| 'waist'     // талія
| 'hips'      // стегна / сідниці
| 'thigh'     // стегно (бедро)
| 'biceps'    // біцепс
| 'neck'      // шия
| 'calf'      // литка
| 'bodyfat'   // % жиру (0–60)
```
Зберігаються так само, як `weight`/`pulse` (HealthEntry, `lastForDay` = останнє за день). Жодної міграції.

### 2.2. Похідні метрики (utils, без нового сховища)
- **ІМТ** — є (`calcBMI`).
- **WHtR** (талія/зріст): `waist / heightCm`; < 0.5 — здорово (універсальний маркер ризику).
- **WHR** (талія/стегна): `waist / hips`; норма ч < 0.9, ж < 0.85.
- **% жиру (оцінка US Navy)** — без розумних терезів, з обводів: `estimateBodyFatNavy(sex, heightCm, neck, waist, hips?)`. Дає orientовну цифру, якщо користувач не вводить `bodyfat` вручну.
- **Суха маса** = `weight × (1 − bodyfat/100)`.

### 2.3. Нагадування — розширення моделі
```ts
interface Reminders {
  water: boolean; sleep: boolean;       // існують
  weight: boolean;                      // НОВЕ — щодня
  measurements: boolean;                // НОВЕ — щотижня
}
```
- `weight` → щоденне нагадування (напр., 08:00).
- `measurements` → **щотижневе** (напр., неділя 09:00) — заміри частіше не мають сенсу.
- Потрібен `scheduleWeeklyReminder(key, weekday, hour, minute, title, body)` (тригер `WEEKLY`).

## 3. UX / екрани

### 3.1. Рекомендований варіант — окремий розділ «Заміри тіла»
Нова плитка на хабі **«Заміри тіла»** (icon `ruler`, колір `#8B5CF6`) → екран `app/health-body.tsx`:

```
┌─────────────────────────────────────────┐
│  ← Заміри тіла              [＋ запис]    │
├─────────────────────────────────────────┤
│  ЗВЕДЕННЯ                                 │
│  Вага 80.4кг  ІМТ 23.1  %жиру ~18         │
│  WHtR 0.46 ✅   WHR 0.88                   │
├─────────────────────────────────────────┤
│  Талія 84 ▾   ▁▂▃▄  −2 см / міс            │
│  Стегна 98    ▃▃▃▄                         │
│  Груди 102    ▄▄▄▄                         │
│  Біцепс 36 ▴   ▂▃▄▅                         │
│  Стегно 58 · Шия 39 · Литка 38            │
└─────────────────────────────────────────┘
```
- Зведення: вага/ІМТ/%жиру + WHtR/WHR з кольоровим статусом (норма/ризик).
- Список вимірів: останнє значення + мінітренд + дельта за період.
- Тап по метриці → деталь із повним графіком (реюз `MiniBarChart`/новий line-chart).

**Вага** показується і тут (первинний ввід), і в зведенні хаба — це ті самі `weight`-записи. «Показники тіла» (`/health-vitals`) лишається для **пульсу** (і майбутнього тиску); щоб не дублювати, на хабі плитку `vitals` звузимо до пульсу/тиску, а вагу/ІМТ винесемо в «Заміри тіла».

### 3.2. Комбінована форма «Запис тіла» (ключове)
Нижній лист `BodyEntrySheet`:
- Поля: Вага, Талія, Стегна, Груди, Біцепс, Стегно, Шия, Литка, % жиру — **усі опційні**.
- **Префіл** останніми значеннями (зручно коригувати лише змінене).
- На «Зберегти» → створює по одному `HealthEntry` на кожне заповнене поле (одна дата).
- Перемикач одиниць пізніше (v1 — метрична: см/кг).

### 3.3. Альтернатива
Розширити наявний `health-vitals.tsx` секціями (Вага+Заміри / Пульс) — менше плиток, але екран великий. **Рекомендую окремий екран** (чистіше, місце під фото/тренди).

## 4. Нагадування про внесення даних

- Секція «Нагадування» на екрані «Заміри тіла» (і/або в Профілі):
  - **Вага — щодня** о 08:00 (перемикач).
  - **Заміри — щотижня** (неділя 09:00) (перемикач).
- Реалізація: `health_reminders` розширюється; хук `setReminder` узагальнюється (daily для weight, weekly для measurements); `notifications.ts` отримує `scheduleWeeklyReminder`.
- **v1:** перемикачі з розумними дефолтами часу. **v2 (опц.):** вибір часу/дня (потребує date/time-picker — `@react-native-community/datetimepicker`, поки немає).

---

## 5. Що ще варто контролювати (моя рекомендація)
- **Талія** — найважливіший для здоров'я обвід (WHtR).
- **Шия** — потрібна для оцінки % жиру US Navy (без терезів).
- **% жиру** — вручну (розумні терези) або авто-оцінка US Navy.
- Похідні: **WHtR**, **WHR**, **суха маса**, ІМТ.
- **Прогрес-фото** (до/після) — сильна фіча, але потребує `expo-image-picker` → **відкласти** (Фаза 5).
- **Тиск** (систолічний/діастолічний) — раніше відкладали; можна додати тут як vital. **Відкласти/окремо.**
