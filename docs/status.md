## Загальна оцінка: 🟡 T2 — Beta (функціонально насичено, потребує стабілізації)

Додаток має широкий набір модулів і продуманий UI, але **не пройшов наскрізного тестування**, частина критичних місць (синхронізація, realtime, crypto, типобезпека) не верифікована на пристрої. До продакшну — стабілізація + QA. Детально за рівнями готовності та продуктовими tiers — у `analysis/READINESS_AND_TIERS.md`.

## Ключові ризики

> **Етап 1 (стабілізація) виконано** для «Спільне» — див. нижче.

1. ✅ **`crypto.randomUUID()` у `shared.tsx`** — ВИПРАВЛЕНО: безпечний `utils/uuid.ts` + полефіл `react-native-get-random-values` підключено на старті (`app/_layout.tsx`).
2. ✅ **Realtime у «Спільне»** — ПОМ'ЯКШЕНО: додано polling-fallback (25с) + індикатор з'єднання («офлайн»). WS збережено як опортуністичний (не валить).
3. ✅ **Конфлікти даних у «Спільне»** — ВИПРАВЛЕНО: `applyRemoteItem` тепер last-write-wins за `updated_at` (не затирає свіжіші локальні).
4. 🟡 **`crypto.subtle.digest` у `meetings.tsx:78`** — Web Crypto `subtle` відсутній у Hermes. НЕ краш (обгорнуто в try/catch), але PKCE для OAuth деградує до **plain** (зниження безпеки). Фікс — `expo-crypto` (`Crypto.digestStringAsync`).
5. 🟡 **Типи/тести не запущені локально** — `node_modules` відсутні в середовищі агента. Додано `jest` + unit-тести утиліт, скрипти `typecheck`/`test`, CI-workflow. **Дія:** `npm install && npm run typecheck && npm run lint && npm test`.
6. 🟠 **Хардкод-рядки в `shared.tsx`** — не через i18n (нема EN) — Етап 2.
7. 🟢 **Множинні екземпляри `useHealthEntries`** — ПОМ'ЯКШЕНО: HealthKit init/sync тепер раз за сесію (модульний прапор); дані доходять до екранів через сховище. Повний провайдер — опційний follow-up.
8. 🟡 **Дати у профілактиці — текстом** (нема date-picker), звіт лікарю — текстовий (нема `expo-print`).

## Що зроблено нещодавно

**Редизайн «Здоров'я»:** хаб зі зведеною статистикою + плитки розділів (історія — модалкою); 6 розділів окремими екранами; профілактика (ліки/огляди/щеплення/звички) з нотифікаціями та експортом звіту лікарю; профілактичні під-екрани — **нижній лист** (`formSheet`); персональні цілі (TDEE/білок/вода), розділення калорій, ІМТ; **тижневі інсайти** на хабі (Етап 3).

**Спільне:** додано глобальний полефіл `react-native-get-random-values` (`_layout`) — захист від crypto-крашів. Виявлено й **видалено застарілий дубль `app/shared.tsx`** (конфлікт маршруту `/shared`); живий таб `(tabs)/shared.tsx` уже має зрілу реалізацію (Clipboard/i18n/групи). *(Прим.: рання «стабілізація Спільне» була застосована до дубліката — поза живим табом.)*

**Якість:** typecheck-гейт виявив і виправлено 10 латентних TS-помилок; lint — 3 помилки виправлено; jest без watchman.

## Рекомендований шлях до Production

1. Встановити залежності → `npx tsc --noEmit` + `expo lint`, виправити типи.
2. Полагодити критичні ризики «Спільне» (crypto, realtime, конфлікти).
3. Пройти тест-плани (`testing/`) на iOS та Android.
4. Винести спільну логіку, прибрати хардкод-рядки, додати EN-переклади де бракує.
5. Реліз-кандидат → бета-тест → продакшн.
