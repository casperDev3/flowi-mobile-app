# Flowi — детальний аналіз готовності + продуктові tiers

**Дата:** 2026-06-29
**Призначення:** оцінити зрілість кожного модуля за рівнями готовності (release-tiers), визначити блокери до продакшну та спроєктувати монетизаційні tiers (Free / Plus / Pro).

---

## Частина I. Рівні готовності (release-tiers)

| Tier | Назва | Критерій |
|------|-------|----------|
| **T0** | Concept | лише ідея/каркас |
| **T1** | Alpha | основний happy-path працює; не тестовано; відомі діри |
| **T2** | Beta | функціонально повний; часткове тестування; є відомі проблеми |
| **T3** | RC (Release Candidate) | блокери закриті; протестовано iOS+Android; поліш завершено |
| **T4** | Production | у проді; стабільно; є моніторинг/аналітика |

**Загальний вердикт додатку: 🟡 T2 (Beta).** Функціонально насичений, але без наскрізного тестування, типобезпека не верифікована, нема автотестів і моніторингу.

## Частина III. Наскрізні quality gates (рівень додатку)

| Gate | Стан | Деталі / дія |
|------|------|--------------|
| **TypeScript** | 🔴 не верифіковано | `node_modules` відсутні в середовищі; новий код здоров'я/профілактики/спільне не прогнано. **Дія:** `npm i && npx tsc --noEmit` |
| **Lint** | 🔴 не запущено | `expo lint` |
| **Автотести** | 🟡 каркас додано | `jest-expo` + unit-тести утиліт (`dateUtils`, `healthUtils`, `preventionUtils`, finance/task) у `__tests__/`. **Дія:** `npm test`; розширити на компоненти/smoke |
| **i18n** | 🟢 значною мірою | Здоров'я/Профілактика/Спільне — повністю (uk+en). Можливі залишки в інших екранах — звірити |
| **Доступність (a11y)** | 🟡 розпочато | `accessibilityLabel`/role на back/close + add-кнопках усіх health-екранів. **Дія:** доповнити решту іконкових контролів + динамічний шрифт |
| **Продуктивність** | 🟠 ризики | Великі екрани (index 3400, health розбито ✅); агрегати O(n) на кожному рендері. **Дія:** профілювання на 1000+ записів |
| **Безпека** | 🟠 | PKCE plain (meetings); спільні групи — лише секрет (нема акаунтів); секрети у відкритому сховищі |
| **Офлайн / помилки** | 🟡 | Спільне — індикатор+банер ✅; інші — переважно тихі catch у dev-логи |
| **Crash-safety** | 🟢 | `crypto.randomUUID` виправлено; `crypto.subtle` — у try/catch (не краш) |
| **Сховище/міграції** | 🟢 | `initialized`-патерн скрізь; health-міграція calories→calories_out |
| **CI/CD, моніторинг, аналітика** | 🟡 каркас додано | CI-workflow (`.github/workflows/ci.yml`: typecheck·lint·test); абстракції `utils/reporting.ts` (Sentry-ready) + `utils/analytics.ts`. **Дія:** підключити реальний Sentry DSN + аналітику-провайдер |

## Частина IV. Блокери до Production (release-gates)

### Gate A — до RC (T3)
1. 🔴 `npm install && npx tsc --noEmit && expo lint` — 0 помилок.
2. 🔴 Пройти тест-плани (`testing/TEST_PLAN_SHARED_HEALTH.md`) на iOS+Android, dark+light.
3. 🔴 Перевірити **HealthKit на фізичному iPhone** і **доставку нотифікацій**.
4. 🔴 «Спільне»: підтвердити роботу бекенду (sync/realtime) або задокументувати обмеження.
5. 🟠 Базові unit-тести утиліт + smoke-запуск кожного екрана.

### Gate B — до Production (T4)
6. 🔴 Crash-репортинг (Sentry) + базова аналітика подій.
7. 🟠 PKCE через `expo-crypto`; перегляд безпеки спільних груп.
8. 🟠 Політика приватності/обробки даних (HealthKit, спільний сервер).
9. 🟡 App Store / Play метадані, скріншоти, опис.

## Частина V. Продуктові tiers (монетизація)

Модель **freemium**: базовий трекінг безкоштовний (utility-hook для утримання), розширена аналітика/колаборація/AI — платні.

### Опис tiers

| | **Free** | **Plus** (~$2.99/міс) | **Pro** (~$5.99/міс або $39/рік) |
|---|----------|------------------------|----------------------------------|
| Позиціонування | Особистий трекер | Здоров'я+родина серйозно | Power-user / AI-коуч |
| Ціль | Залучення/утримання | Основна конверсія | Максимальна цінність |

### Матриця функцій

| Функція | Free | Plus | Pro |
|---------|:----:|:----:|:---:|
| Завдання, Час, Нотатки, Проєкти, Контейнери | ✅ | ✅ | ✅ |
| Фінанси (базові транзакції/категорії) | ✅ | ✅ | ✅ |
| Здоров'я базове (вода/кроки/сон/вага/пульс) | ✅ | ✅ | ✅ |
| HealthKit-синк | ✅ | ✅ | ✅ |
| **Персональні цілі (TDEE/білок/вода)** | — | ✅ | ✅ |

| **Тижневі інсайти** | 1 інсайт | ✅ усі | ✅ + 3-міс тренди |
| **Спільні групи** | 1 група, до 2 учасників | необмежено | необмежено |

### Конверсійні гачки (де показувати upsell)
- Спроба додати **2-гу спільну групу** → Plus.
- Відкриття **макросів/персональних цілей** у Харчуванні → Plus.
- Кнопка **«Експорт звіту лікарю»** → Pro.
- Блок **інсайтів** «розблокувати всі тренди» → Plus/Pro.
- Налаштування **синхронізації профілю** → Pro.

### Метрики на tier
- Free→Plus конверсія (ціль 3–5%), Plus→Pro апсейл, trial→paid, churn по tier, ARPU.

## Частина VI. Дорожня карта за tiers готовності

| Спринт | Фокус | Вихід |
|--------|-------|-------|
| 1 | Quality gates A (tsc/lint/тести) + device-QA | Здоров'я/Спільне → **T3** |
| 2 | Бекенд «Спільне» (realtime/conflict на сервері) + Sentry/аналітика | Спільне → **T3**, інфраструктура → готова |
| 3 | Пакето-залежні: `expo-crypto` (PKCE), date-picker'и, QR; a11y-мітки | Профілактика/Зустрічі → **T3** |
| 4 | Монетизація: paywall + tiers-гейти (Free/Plus/Pro) + білінг | Реліз-кандидат усього → **T3→T4** |
| 5 | Pro-фічі: PDF-звіт, AI-поради, синхр. профілю | Pro-цінність → **T4** |

## Підсумок

- **Зараз:** додаток на **T2 (Beta)** — багато функцій, але без верифікації якості та інфраструктури запуску.
- **Найближчий блокер:** прогнати `tsc`/`lint`/тести та device-QA (Gate A) — це підіймає ключові модулі до **T3**.
- **Монетизація:** чітка freemium-модель Free/Plus/Pro з природними гачками (колаборація, макроси, звіт лікарю, AI).
- **Критичний борг:** автотести, crash-моніторинг, аналітика — їх відсутність робить будь-який реліз сліпим.
