## 3. Негативні / межові сценарії

| ID | Сценарій | Очікуваний результат |
|----|----------|----------------------|
| N-01 | Порожній ввід у будь-якій модалці → «Зберегти» | Дія ігнорується (без падіння, без порожнього запису) |
| N-02 | Некоректний час ліків («25:99», «abc») | Парситься лише валідний `HH:MM`; некоректне ігнорується |
| N-03 | Некоректна дата огляду («2026-13-40») | Не валить; запис із безпечним розбором дати |
| N-04 | Вимкнені нотифікації (Deny) | Перемикачі/планувальники не валять застосунок; стан коректний |
| N-05 | Сервер «Спільне» 500/timeout | Тихий фейл sync без краху; (після фіксу) банер помилки |
| N-06 | Великий обсяг записів здоров'я (1000+) | Агрегати/чарти рахуються без помітних лагів |
| N-07 | Швидкі повторні тапи «+»/чіпів | Без дубль-записів/гонок |
| N-08 | Перемикання вкладок під час синку | Без витоків WS/таймерів |
