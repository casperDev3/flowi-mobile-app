# Flowi — Аналіз застосунку та рекомендації

## Виправлення (Bugs / UX Issues)

### Критичні

1. **`toDateString()` у групуванні даних** — `workouts.tsx`, `health.tsx`, `notes.tsx` та інших екранах використовують `new Date(w.date).toDateString()` для групування, що може спричинити timezone-баги. Замінити на `isSameDay()` з `utils/dateUtils.ts`.

2. **`initialized` прапор відсутній у деяких екранах** — Деякі Stack-скріни зберігають дані у `useEffect` без перевірки `initialized`, що може перезаписати AsyncStorage порожнім масивом при першому рендері.

3. **Відсутня обробка помилок AsyncStorage** — `loadData`/`saveData` ловлять помилки мовчки. Потрібно принаймні логувати в dev-режимі та показувати toast-повідомлення користувачу.

4. **`workouts.tsx` — відсутній `headerShown: false`** — Стандартний Stack-хедер перекривав кастомний. Виправлено через `<Stack.Screen options={{ headerShown: false }} />`.

### Середні

5. **Зустрічей (meetings.tsx) немає навігації з головного екрану** — Екран зустрічей не посилається ні з однієї вкладки. Потрібна точка входу.

6. **Екран `time.tsx` з `href: null` недоступний без контексту** — Якщо `pendingTask` не встановлено, перехід на вкладку часу неможливий. Потрібна альтернативна кнопка навігації.

7. **Pull-to-refresh відсутній** на деяких Stack-екранах (notes, projects, ideas).

8. **Немає підтвердження при масовому видаленні** — `data.tsx` очищає всі дані без фінального підтвердження.

9. **`time_entries` не мають `initialized` прапора** — Може спричинити очищення записів часу при першому завантаженні.
