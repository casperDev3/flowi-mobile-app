# План: режим Офлайн / Онлайн (приватний локальний режим)

**Дата:** 2026-06-30
**Запит:** додати перемикач режимів Офлайн/Онлайн. В офлайн — дані лише на пристрої, онлайн-функції недоступні (екран заблюрено + «Недоступно в офлайн-режимі»). В онлайн — онлайн-функції доступні.

---

## 1. Аналіз ідеї

### 1.1. Поточний стан
Додаток **офлайн-first**: усі дані (завдання/фінанси/здоров'я/час/нотатки…) вже зберігаються **локально** в AsyncStorage і повністю працюють без мережі. «Онлайн» — це **надбудова** поверх локальних даних.

**Онлайн-точки (потребують мережі):**
| Функція | Файл | Що робить онлайн |
|---------|------|------------------|
| **Спільне** (колаборація) | `app/(tabs)/shared.tsx` + `store/api-config.ts` | Групи, sync через сервер, polling/WS |
| **Синхронізація пристроїв** | `app/sync.tsx` + `store/sync.ts` + `store/supabase.ts` | Хмарна синхронізація між пристроями |
| **AI-агент** | `app/(tabs)/agent.tsx` | Запити до LLM (онлайн) |
| **Зустрічі → Google Calendar** | `app/meetings.tsx` | OAuth + імпорт подій |
| **Надіслати девелоперу** (баги/ідеї) | `app/bugs.tsx`, `app/ideas.tsx` | POST на Google Apps Script |
| Авто-бекап | `store/auto-backup.tsx` | **ЛОКАЛЬНИЙ файл** на пристрої → офлайн-сумісний ✅ |
| Нотифікації | `store/notifications.ts` | Локальні → офлайн-сумісні ✅ |

### 1.2. Важливе уточнення
«Зберігати дані лише на пристрої» — це **вже так за замовчуванням**. Тож офлайн-режим фактично = **вимкнути всі мережеві функції/синки** й заблокувати онлайн-екрани. Онлайн-режим = дозволити їх поверх локальних даних.

### 1.3. Цінність
- **Приватність** як фіча: «дані ніколи не покидають пристрій» — сильний маркетинговий і довірчий аргумент (особливо для здоров'я/фінансів).
- **Контроль і передбачуваність:** нуль фонових мережевих запитів, економія трафіку/батареї.
- **Монетизація:** офлайн — безкоштовно/приватно; онлайн-колаборація/sync — Premium (узгоджується з tiers у `analysis/READINESS_AND_TIERS.md`).

### 1.4. Ризики / нюанси
- **Що блокувати повністю vs частково.** Суто онлайн-екрани (Спільне, Sync, Agent) — повний оверлей. Змішані (Зустрічі — є локальні зустрічі + онлайн-імпорт; Баги/Ідеї — локальне зберігання + онлайн-надсилання) — краще **вимикати лише онлайн-дії**, не блокуючи корисний офлайн-функціонал.
- **Перемикання режимів.** Офлайн→Онлайн: за бажанням запропонувати синхронізацію. Онлайн→Офлайн: зупинити активні WS/polling/синки.
- **Ручний режим vs реальна мережа.** Запит — про **ручний** перемикач. Реальну відсутність мережі (`@react-native-community/netinfo`) можна додати окремо як авто-індикатор (не блокувати, а підказувати). У v1 — ручний режим.
- **Локальні «онлайн-подібні» речі лишаються:** локальний авто-бекап, експорт/імпорт файлу, нотифікації — працюють офлайн.

## 2. Цільова поведінка

- Глобальний **режим**: `online | offline`, збережений у сховищі (`app_mode`), за замовч. — **online** (не змінює поточну поведінку; користувач свідомо вмикає офлайн).
- **Офлайн:**
  - Жодних мережевих викликів/синків (shared sync/WS, device sync, AI, OAuth, reporter).
  - Суто онлайн-екрани → **заблюрений контент + картка «Недоступно в офлайн-режимі»** + кнопка «Увімкнути онлайн».
  - Змішані екрани → онлайн-кнопки вимкнено/приховано, локальне лишається.
  - Бейдж-індикатор «офлайн» (напр., у Налаштуваннях і на онлайн-табах).
- **Онлайн:** усе як зараз.
- Перемикач — у **Налаштуваннях** (секція «Режим роботи»), з поясненням про приватність.

## 3. Архітектура

```
store/app-mode.tsx           — AppModeProvider + useAppMode(): { online, setOnline }
                               persist 'app_mode'; модульний кеш isOnlineMode() для не-React місць
components/shared/OfflineOverlay.tsx
                               — <OfflineOverlay>{children}</OfflineOverlay>:
                                 offline → діти під BlurView + центральна картка + кнопка
hooks/ (за потреби)          — useRequireOnline() для дій
```

- **Провайдер** `AppModeProvider` обгортає додаток у `app/_layout.tsx` (поряд з I18n/Theme).
- **Гейтинг мережі:** перед будь-яким `fetch`/sync — перевірка `isOnlineMode()` (або `online` з контексту). Централізувати: у `shared.tsx` (sync/WS), `sync.ts`, `agent`, `meetings` (import), `bugs/ideas` (send).
- **OfflineOverlay** — реюзабельна обгортка для онлайн-екранів.

---

## 4. Що з кожною онлайн-функцією

| Функція | Дія в офлайн |
|---------|--------------|
| Спільне (`shared`) | Повний `OfflineOverlay`; не стартувати sync/WS/polling |
| Синхронізація (`sync`) | Повний `OfflineOverlay`; не викликати supabase |
| AI-агент (`agent`) | Повний `OfflineOverlay` (екран введення заблоковано) |
| Зустрічі (`meetings`) | Локальні зустрічі працюють; кнопку «Імпорт з Google Calendar»/OAuth — приховати/disabled |
| Баги/Ідеї | Зберігати локально як є; кнопку «Надіслати девелоперу» — disabled + підказка |
| Авто-бекап, експорт/імпорт файлу, нотифікації | Працюють (локальні) |

## 5. UX — оверлей «Недоступно офлайн»

```
┌───────────────────────────────┐
│  (контент екрана — заблюрено)  │
│        ┌───────────────┐       │
│        │   ☁︎⃠  іконка    │       │
│        │ Недоступно в    │       │
│        │ офлайн-режимі   │       │
│        │ [ Увімкнути     │       │
│        │   онлайн ]      │       │
│        └───────────────┘       │
└───────────────────────────────┘
```
- `BlurView` поверх контенту (`intensity` ~30) + напівпрозорий шар.
- Іконка (напр., `wifi.slash` / `icloud.slash`), заголовок, підпис, кнопка «Увімкнути онлайн» → `setOnline(true)`.

| Фаза | Зміст | Файли |
|------|-------|-------|

| **2. Перемикач** | Секція «Режим роботи» в Налаштуваннях + пояснення/приватність | `app/(tabs)/settings.tsx`, i18n |

## 8. Edge cases / рішення
- **Старт у офлайні:** провайдер читає `app_mode` до першого рендера онлайн-екранів; поки не завантажено — вважати online (або показувати лоадер).
- **Перехід онлайн→офлайн під час активного синку «Спільне»:** у `useFocusEffect`/ефектах перевіряти `online`; при false — `clearInterval` + `ws.close()`.
- **Не-React місця** (utils/store) читають режим через `isOnlineMode()` (модульний кеш, оновлюється разом із контекстом).
- **Безпека:** офлайн гарантує нуль вихідних запитів — варто прибрати/гейтнути навіть аналітику/репортинг у офлайні.

---

## 9. Питання для підтвердження
1. **Default-режим:** Online (як зараз) чи Offline (приватність за замовч.)?
2. **Змішані екрани** (Зустрічі, Баги/Ідеї): вимикати лише онлайн-дії (реком.) чи блокувати весь екран оверлеєм?
3. **Бейдж режиму:** показувати індикатор «офлайн» у хедерах/таб-барі чи лише в Налаштуваннях?
4. **Реальна детекція мережі** (netinfo) — зараз чи відкласти (потребує пакета)?
5. **Перехід офлайн→онлайн:** одразу пропонувати синхронізацію чи нічого не робити?

> Після підтвердження почну з **Фази 1** (ядро режиму + провайдер — безпечно, нічого не блокує), далі перемикач, оверлей і гейтинг.
